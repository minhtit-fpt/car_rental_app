2. SQLite trên máy (Drift) — cache & offline

Các bảng này không có FK thật tới server. Cột id = chính id của bản ghi trên Postgres (mirror). Quan hệ nội bộ duy nhất: ChatMessagesLocal nối ChatConversation theo conversationId.

erDiagram
    VehiclesCache {
        text id PK "= Vehicle.id server"
        text json "snapshot"
        real lat
        real lng
        int fetchedAt "TTL 10 phút"
    }
    BookingsCache {
        text id PK "= Booking.id server"
        text status
        text json "snapshot"
        int fetchedAt "TTL 30 phút"
    }
    NotificationsCache {
        text id PK "= Notification.id server"
        text type
        bool readLocal
        int fetchedAt
    }
    SearchHistory {
        int id PK "auto"
        text keyword
        text filters "json"
        int searchedAt
    }
    BookingDrafts {
        int id PK "local"
        text vehicleId "= Vehicle.id"
        int startTime
        int endTime
        bool deliveryWanted
        text syncState "draft|submitting"
    }
    ChatMessagesLocal {
        int localId PK
        text serverId "null khi chưa gửi"
        text conversationId
        text body
        text syncState "pending|sent|failed"
    }
    ChatConvLocal {
        text id PK "= ChatConversation.id"
        text lastBody
        int lastAt
    }

    ChatConvLocal ||--o{ ChatMessagesLocal : "chứa (conversationId)"
Ánh xạ máy ↔ server (theo id, không phải FK ràng buộc):
VehiclesCache.id → Vehicle.id  ·  BookingsCache.id → Booking.id  ·  BookingDrafts.vehicleId → Vehicle.id  ·  ChatMessagesLocal.serverId → ChatMessage.id (sau khi gửi thành công)

3. Ngoài SQLite — 2 kho riêng trên máy

Không dùng SQLite vì lý do bảo mật / đơn giản.

flutter_secure_storage (Keychain iOS / Keystore Android — mã hoá phần cứng): accessToken · refreshToken · userRole. Token không bao giờ để trong file .db.
Hive / SharedPreferences (key-value nhẹ, không quan hệ): onboardingDone · themeMode · locale · lastSyncAt.

4. KHÔNG bao giờ cache — luôn hỏi server

Payment (status, amount) · Contract đã ký · Ảnh KYC (CCCD, bằng lái) · Booking status lúc đang giao dịch · Submit KYC.
Quy tắc vàng: dữ liệu liên quan tiền hoặc pháp lý → luôn lấy từ Postgres, không tin bản cache trên máy.
